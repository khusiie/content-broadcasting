2. AUTHENTICATION FLOW
================================================================================

- User visits / (login page).
- If already authenticated, PublicOnlyRoute redirects to dashboard.
- User fills login form → validated by Zod schema → sent via authService.login().
- authService stores JWT token and user object in localStorage.
- AuthContext updates with user data.
- ProtectedRoute checks AuthContext for authentication and role.
- Token is automatically attached to all API calls via Axios request interceptor.
- Logout clears localStorage and resets AuthContext state.
- Unauthenticated access to protected routes → redirected to /.


3. ROLE-BASED ROUTING
================================================================================

  Teacher:
    /teacher           → TeacherDashboard (stats + recent uploads)
    /teacher/upload    → UploadContent (form with validation)
    /teacher/content   → MyContent (paginated list)

  Principal:
    /principal         → PrincipalDashboard (stats + quick actions)
    /principal/pending → PendingApprovals (approve/reject workflow)
    /principal/content → AllContent (search + filter + paginate)

  Public (no auth):
    /live/:teacherId   → LiveDisplay (active broadcasts)

  Guards:
    - ProtectedRoute checks auth + role
    - PublicOnlyRoute redirects logged-in users to their dashboard
    - Loading state shown during auth check
    - Fallback: all unknown routes redirect to /


4. API INTEGRATION APPROACH
================================================================================

  Architecture:
    Components → Hooks → Services → API (Axios)

  Key Principles:
    - Components NEVER call api directly.
    - All API calls go through service files (services/*.service.js).
    - Custom hooks (hooks/*) wrap services with React state (loading, error, data).
    - Components consume hooks for data and actions.

  Service Layer:
    - auth.service.js:    login(), register(), logout(), getCurrentUser(), getToken()
    - content.service.js: uploadContent(formData), getMyContent(), getLiveContent(teacherId)
    - approval.service.js: getPending(), getAll(params), approve(id), reject(id, reason)

  Error Handling:
    - Services throw on failure; hooks catch and set error state.
    - Components display errors via toast notifications (not alert()).
    - Axios interceptor attaches auth token automatically.

  Replaceability:
    - To swap to a real backend, only service files need updating.
    - No component changes required.


5. STATE MANAGEMENT APPROACH
================================================================================

  Strategy: React Context + Custom Hooks (no external state library)

  AuthContext:
    - Stores: user, loading, isAuthenticated, isTeacher, isPrincipal
    - Actions: login, register, logout
    - Consumed via useAuth() hook

  ToastContext:
    - Stores: toast queue (array of { id, message, type })
    - Actions: success(), error(), info(), warning(), removeToast()
    - Auto-dismiss after configurable duration
    - Consumed via useToast() hook

  Local State:
    - Page-specific state (filters, pagination, modals) kept in component useState
    - Computed values use useMemo for performance
    - Event handlers wrapped in useCallback to prevent unnecessary re-renders

  Performance:
    - React.memo on all reusable components
    - useMemo on filtered lists, stats computation, pagination slicing
    - useCallback on all handler functions
    - Lazy image loading with loading="lazy"


6. FORM HANDLING & VALIDATION
================================================================================

  Library: React Hook Form + Zod (via @hookform/resolvers)

  Schemas (utils/validators.js):
    - loginSchema: email (required, valid format), password (required)
    - registerSchema: name (required, min 2), email, password (min 6), role (enum)
    - uploadContentSchema: title, subject, start_time, end_time (with end > start refinement), rotation_duration
    - File validation: type (JPG/PNG/GIF only), size (max 10MB)

  Features:
    - Real-time validation errors displayed per field
    - File preview before upload via URL.createObjectURL()
    - File size and type validation on selection (before upload)
    - End time must be after start time (Zod .refine())


7. ASSUMPTIONS
================================================================================

  - JWT tokens do not expire during a session (no refresh token flow).
  - The backend API is running at http://localhost:5000/api.
  - File uploads are limited to JPG, PNG, and GIF under 10MB.
  - Rotation duration is specified in minutes.
  - The "live" endpoint returns an array of currently active content items.
  - Scheduling status (Scheduled/Active/Expired) is computed client-side from timestamps.
  - Only one principal and multiple teachers can exist in the system.
  - The public /live/:teacherId page requires no authentication.
  - Content status values: "uploaded", "pending", "approved", "rejected".
